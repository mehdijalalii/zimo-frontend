# Sitemap: Add Explicit Locale URLs

## Problem
Sitemap only lists the default (fa) URL per page. `/en/` and `/tr/` versions are only referenced via `alternates.languages` but not as separate crawlable entries.

## Fix
In `src/app/sitemap.ts`, loop through all locales for each page and generate a separate URL entry per locale, each with its own `alternates.languages` block.

## File
- `src/app/sitemap.ts`

## Verification
- `npm run build` passes
- `/sitemap.xml` shows 3 entries per page (one per locale): `zimo.beauty/`, `zimo.beauty/en/`, `zimo.beauty/tr/`

### 2. Duplicate Meta Descriptions on Main Pages
**Files:** `src/messages/fa.json:6-8`, `en.json:6-8`

### 3. Missing Open Graph Images on All Sub-Pages
**Files:** All page `generateMetadata` functions (features, pricing, faq, about, contact, demo, feature/[slug])

Only the root layout and home page set OG images. All sub-pages (features, pricing, faq, about, contact, demo, feature detail pages) have **no OG images** ŌĆö only `openGraph.title` and `openGraph.description`.

**Fix:** Add `openGraph.images` to every page's `generateMetadata`. Use feature-specific screenshots or a shared branded OG image per locale.

### 5. Pages That Should Be Noindexed
**Files:** `src/app/[locale]/services/page.tsx`, `blog/page.tsx`, `testimonial/page.tsx`, `case-study/page.tsx`, `legal/page.tsx`, `terms/page.tsx`, `privacy/page.tsx`, `login/page.tsx`, `signup/page.tsx`

These are all "coming soon" / placeholder pages with no real content. They should be `noindex` until content is ready. Currently they're fully indexable.

**Fix:** Add `robots: { index: false }` to the `generateMetadata` of each placeholder page.

### 6. Missing Not-Found and Error Pages
**No files found** for `not-found.tsx` or `error.tsx`.

Custom 404 and 500 pages improve user experience and prevent soft-404 issues.

**Fix:** Create `src/app/[locale]/not-found.tsx` and `src/app/[locale]/error.tsx` (or `src/app/not-found.tsx`).

## High Impact Improvements

### 7. Missing Structured Data (JSON-LD) on Most Pages
**Current:** Only `layout.tsx` has SoftwareApplication JSON-LD, and `faq/page.tsx` has FAQPage JSON-LD.

Missing opportunities:
- **Feature pages**: Add `SoftwareApplication` + `FAQPage` for each feature's FAQ section
- **Pricing page**: Add `Product` schema with pricing tiers
- **About page**: Add `Organization` schema
- **Contact page**: Add `ContactPage` schema
- **BreadcrumbList** schema on all sub-pages (breadcrumbs already exist in code but not as structured data)

### 8. Twitter Card Images Missing on Sub-Pages
Same as OG images ŌĆö only the root layout sets `twitter.images`.

**Fix:** Add `twitter.images` to every page's metadata.

### 9. Missing `not-found.tsx` Metadata
Without a custom not-found page, 404s will use default title/description and won't have proper SEO signals.

### 10. Hero Image Missing `width`/`height` on Layout JSON-LD
The JSON-LD in `layout.tsx:84-103` references `offers.price: "0"` and `priceCurrency: "IRR"` ŌĆö the default locale is `fa` (Iranian Rial). This is fine for local SEO but should be validated.

### 11. Internal Linking Gaps
- No links from feature pages to each other (no "related features" section)
- Footer links are good but could add links to blog/services when ready
- No breadcrumbs on main pages (only on feature detail pages)

### 12. robots.txt Could Be More Specific
**File:** `src/app/robots.ts`

Currently blocks `/api/` and `/_next/`. Should also explicitly:
- Allow all common bot paths
- Consider disallowing `/login`, `/signup` paths
